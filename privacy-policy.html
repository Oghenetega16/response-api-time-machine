1 What the Extension Does

API Response Time Machine is a developer tool that intercepts and locally records JSON API responses made by your browser. It is designed to help developers detect when an API's response structure changes silently over time, and to compare any two captured responses side-by-side.

The extension only activates when you are using your browser. It does not run in the background when Chrome is closed. Recording can be paused at any time from the extension popup.

2 What Data Is Captured

When recording is enabled, the extension captures the following information for each qualifying network request made by your browser:

Field Example Purpose
Request URL https://api.example.com/users/1 Identifies the endpoint. Used as the key for shape change detection.
HTTP method GET, POST, PUT, PATCH, DELETE Part of the endpoint identifier used for grouping and diffing.
HTTP status code 200, 404, 500 Displayed in the Feed for quick diagnosis.
Response body {"id": 1, "name": "Alice"} The JSON response from the server. Stored up to 50 KB per response. Truncated if larger.
Request body
(POST/PUT/PATCH only)		 {"name": "Alice"} The request payload you sent. Stored up to 20 KB. Truncated if larger. Only captured for mutation requests.
Response headers Content-Type: application/json Displayed in the Headers tab of the record detail view.
Request headers Authorization: Bearer … Displayed in the Request tab of the record detail view. Note: this includes any Authorization or Cookie headers sent with the request.
Response shape {"id": "number", "name": "string"} A structural schema derived from the response. Field names are preserved; all values are replaced with their type name. Used for change detection.
Timestamp Unix milliseconds When the response was received. Used for timeline display.
⚠️
Authorization headers and sensitive request bodies: The extension captures request headers, which may include Authorization tokens, Cookie headers, and API keys. It also captures request bodies for POST/PUT/PATCH requests, which may contain sensitive data. This data is stored only on your local device and is never transmitted anywhere, but you should be aware it is recorded. Use the URL filter in Settings to exclude sensitive endpoints from capture.

What is not captured:

  • Non-JSON responses (HTML pages, images, binary data, plain text) — these are silently ignored
  • WebSocket frames
  • Responses from URLs matching your exclude patterns (configurable in Settings)
  • Anything captured while recording is paused
  • Browser history, bookmarks, tabs, or page content outside of network requests
  • Keystrokes, form input, clicks, or any user interaction on the page

3 Where Data Is Stored

All captured data is stored exclusively in chrome.storage.local — a sandboxed storage area managed by Chrome that is tied to your browser profile on your local device. This storage is:

  • Local only — it exists only on the device where you installed the extension
  • Not synced — it is not uploaded to Chrome Sync or Google's servers
  • Not shared — it is not accessible to any website or other extension
  • Not transmitted — the extension makes zero outbound network requests of any kind

The following storage keys are used:

Storage Key Contents Size Limit
atm_records The full list of captured response records (up to 5,000). Oldest records are automatically pruned when the limit is reached. 5,000 records maximum
atm_shape_history Per-endpoint shape change history. Stores the structural schema for each distinct shape version an endpoint has returned. Preserved independently of the record cap. 50 change entries per endpoint
atm_config Your settings: whether recording is enabled, URL include/exclude filters, and notification preferences. Contains no personal data. Negligible
✅
You can delete all stored data at any time. Open the extension popup → Settings → Clear All Records. This permanently removes all records and shape history from chrome.storage.local. You can also delete individual records from the record detail modal.

4 Data You Choose to Export

The extension provides two export formats: JSON (native format including shape history) and HAR (HTTP Archive, compatible with Chrome DevTools, Postman, and Insomnia). These exports are created as files downloaded to your local device.

Exported files contain the same data listed in Section 2, including any request headers, authorization tokens, and request bodies that were captured. Treat exported files as sensitive. Do not share them with untrusted parties, as they may contain authentication tokens or API keys from your captured sessions.

The extension can also import previously exported JSON files to restore records. Import data is merged into chrome.storage.local and never transmitted anywhere.

5 Chrome Permissions and Why They Are Needed

The extension requests the following Chrome permissions. Each is required for the core functionality described below and is used for no other purpose.

Permission Why it is needed What it does NOT do
<all_urls>
(host permission) 		The extension must observe and intercept network activity on any domain you browse to. API calls happen on every website — restricting to specific domains would make the tool useless for monitoring third-party APIs. Does not read page content, modify requests, or interact with websites in any way other than intercepting JSON API responses.
webRequest Used to observe response metadata (URL, method, status code, response headers) for each network request. This is how the extension knows a request completed and can associate metadata with the response body captured by the page-level script. Does not modify or block any requests. The extension uses read-only observation only.
scripting Required to inject injected.js into the page's main JavaScript context. Chrome content scripts run in an isolated world and cannot access window.fetch — the injected script must run in the page's own context in order to wrap fetch and XMLHttpRequest and capture response bodies. Does not execute arbitrary scripts on demand. The only injected script is the static injected.js file bundled with the extension, which exclusively wraps fetch and XHR to capture JSON responses.
storage Required to read and write captured records, shape history, and settings to chrome.storage.local. Does not access chrome.storage.sync. All storage is local only.
alarms Manifest V3 service workers are killed by Chrome after approximately 30 seconds of inactivity. A repeating alarm fires every 24 seconds to wake the service worker and re-register the webRequest listener, ensuring no responses are missed during a recording session. Does not schedule any actions other than the internal service worker keepalive.
tabs Used to identify which tab a captured response originated from, so the extension can display the correct host information in the Feed. Does not read tab URLs, titles, or history. Does not track browsing behaviour.
notifications Used exclusively to show a browser notification when an API response shape change is detected — and only when you have explicitly enabled this feature in Settings. Notifications are off by default. Does not show notifications for any purpose other than shape change alerts that you have opted into.

6 Third-Party Services

The extension uses no third-party services of any kind. It does not integrate with analytics platforms, error monitoring services, crash reporters, advertising networks, or any external API. There is no backend server. No data is ever transmitted outside of your local device.

The extension loads fonts from Google Fonts in the popup UI (fonts.googleapis.com). This is the only external connection the popup makes — it is a standard font stylesheet request and does not transmit any captured API data.

ℹ️
If you want to verify the extension makes no data-exfiltration requests, you can inspect its network activity in Chrome DevTools by opening the extension's service worker from chrome://extensions → "Service Worker" and observing the Network tab while using the extension.

7 Children's Privacy

API Response Time Machine is a developer tool intended for adults. It is not directed at children under the age of 13, and we do not knowingly collect personal information from children. Since the extension collects no personal information at all (all data stays local on your device), it does not collect children's personal information either.
